What：计算机程序设计（英语：Computer programming），或称程序设计（coding），是给程序解决出特定问题的过程，软件开发过程中的重要步骤。程序设计方法往往以某种程序设计语言为工具，给出这种语言下的程序。(from Wikipedia)

Who： 秦逸(南京大学仙林校区计算机科学与技术系楼501室；)

Where：鼓楼校区新教-107（理论课）/南园综合楼503（上机课）

When：每周周一5-6节（理论课)/7-8节（上机课），单周周三5-6节（理论课）

Which：《Python程序设计》，张莉主编，高等教育出版社，2022年第二版;《Python程序设计实验指导》，张莉、陶烨编，高等教育出版社，2022年第一版.

Why：程序设计是高效利用计算机的基础。Python语言是目前应用最为广泛的程序设计语言之一，其轻量级和易用性使得其成为数值计算和人工智能领域的主导性语言之一。

课程规则

课程平时成绩占总成绩的40%，主要包括课后作业、期中小测和期末课程项目；期末考试成绩占总成绩的60%，形式为闭卷的上机测试。

平时成绩主要包括：1）课后作业（即上机实习课布置的编程题）；2）期中小测（会在学期中段进行，形式类似于期末考试，但是题量和难度会下降）；3）课程项目（团队形式，可以自由组队、自由选题）。其中课后作业主要根据提交情况给分，我们鼓励同学以功能正确的程序为目标提交自己的课后作业，但是考虑到不同同学的基础以及对于程序算法的掌握程度，因此在评定此部分分数的时候仅以同学们的提交记录为依据；期中小测则会根据提交解答的正确性给出百分制的评分；课程项目则会以随堂汇报答辩的形式进行，每个团队需要准备一个不超过5min的ppt，用于说明自己团队所完成项目的功能目标、主要程序结构以及团队分工与协作状况。

期末考试会在学期结束后进行，形式为闭卷上机试验，主要考察课程讲述的Python语言的基本使用方法，主要题型包括程序阅读题、程序填空题、程序改错题和编程题。

期中小测说明：期中小测将于11月17日下午的7-8节课随堂进行，形式为闭卷上机考试。请所有同学到场参加考试。

测试范围：第1章~6.3的例题6.4之前（不包括6.2）。

测试流程：使用TeachingA系统/万维考试平台。考试启动后，监考老师会请大家登录。用学号登录，登录后检查是否是自己的姓名。所有答卷都必须保存在试题文件夹（默认用IDLE打开的文件就在该文件夹内，答完题目直接保存即可），否则提交不到服务器。测试需要提交两次解答，第一部分一次，第二部分一次。同时需要自备空白草稿纸。

测试题型：参考题型。第一部分是阅读程序题，全屏考试不允许用Python环境，此部分需要单独提交，且提交后无法修改。第二部分题型为完善程序、改错、编程三种，可以使用Python环境，系统默认使用IDLE，如果要用spyder的同学可以把题目拷贝出来答题，但答完后需要再拷贝回去保存。不建议直接用打开方式选择spyder(因为系统中中文支持的不好会乱码）。

成绩查询：查询入口。请大家用自己的学号登录NJUTtable进行查询，注意此处的分数均为未经过任何调整的卷面分数，并不完全对应于各自总评分数中的期中占比部分。

期末考试说明：

期末考试的练习系统已经开放，时间为即日起到12月28日前。可以使用练习模式熟悉考试系统，并查看模拟题目的参考答案。

期末考试范围为1-10章的全部内容，具体以课本内容为基础，不包括老师在课堂上补充的内容（如计算机程序的语义，高级算法设计，关系代数等）。

作业缴纳情况查询：查询入口。请大家用自己的学号登录NJUTtable进行查询。由于部分同学未按照要求（以自己的学号作为注册用户名）注册OJ系统，因此上述作业缴纳统计存在不准确的情况。请所有缴纳了作业但统计次数不对的同学于12月31日考试后找老师说明并查询相关情况，如果当天未找老师，也可在考试后一周内通过邮件与老师联系说明相关情况。同时如果有同学未按时缴纳作业，可在2026年1月12日23：59前将对应的作业文件发送到老师邮箱。注意补交的作业最多会计算成1次正常的缴纳，即如果“只缴纳了2次作业”，那么补交作业会将缴纳情况修改“缴纳3次作业”。

请所有成功在系统内申请了免修不免考同学于考试后一周内通过邮件与老师联系说明相关情况，提醒老师按照免修不免靠政策给出总评分数。
